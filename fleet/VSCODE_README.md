## Basic How-To's

### Start Claude Code
- **Keyboard:** `Ctrl+Shift+P` → type "Claude" → select "Claude: Open"
- **Terminal:** Open terminal (`Ctrl+``) → type `claude`

### Start Gemini CLI
- **Terminal:** Open terminal → type `gemini`
- **Toggle agent mode:** `/agent-mode` (switches between approval-gated and autonomous writes)

### Check Fleet Status
```bash
python fleet/lead_client.py status
```

### Dispatch a Task to Agents
```bash
# Natural language — BigEd figures out the skill
python fleet/lead_client.py task "review the security of fleet/providers.py"

# Explicit skill dispatch
python fleet/lead_client.py dispatch --skill code_review --payload '{"file": "fleet/worker.py"}'
```

### Respond to Agent Questions
Agents sometimes pause and ask for human input (HITL). Check and respond:
```bash
# See what agents are asking
curl http://localhost:5555/api/tasks/waiting-human

# Respond to a specific question
curl -X POST http://localhost:5555/api/tasks/123/respond \
  -H "Content-Type: application/json" \
  -d '{"response": "Yes, approved"}'
```

### See Task Results
```bash
python fleet/lead_client.py result <task_id>
```

## How Prompts Distribute Work

When you type a prompt in Claude Code or Gemini CLI, the work splits between **your VS Code session** and **fleet agents**:

### What Claude Code / Gemini Does (your session):
- **Direct code edits** — modifying files you're looking at
- **Debugging** — reading errors, fixing bugs in real-time
- **Architecture decisions** — planning, designing, choosing approaches
- **Interactive review** — discussing code with you, answering questions

### What Fleet Agents Do (background):
- **Bulk operations** — reviewing 10 files, scanning for security issues
- **Research** — web search, paper fetching, knowledge synthesis
- **Quality checks** — code quality analysis, regression detection
- **Evolution** — idle agents self-improve skills when queue is empty
- **Monitoring** — Dr. Ders watches thermals, VRAM, model health

### How to Delegate to Agents:
```bash
# "Hey fleet, review all the skills"
python fleet/lead_client.py task "review code quality across all skills in fleet/skills/"

# "Run a security audit"
python fleet/lead_client.py task "security audit of the fleet codebase"

# "Summarize recent research"
python fleet/lead_client.py task "summarize the last 5 arxiv papers on RAG"
```

### How Work Flows Back:
1. You dispatch a task → it enters the PENDING queue
2. An available agent claims it → status becomes RUNNING
3. Agent completes → result stored in `fleet/knowledge/`
4. If agent needs your input → WAITING_HUMAN (check Fleet Comm in BigEd)
5. You can check results: `python fleet/lead_client.py result <task_id>`

## Fleet API Reference

Dashboard runs at `http://localhost:5555` while BigEd is active.

### Most Used Endpoints

| Endpoint | Method | What |
|----------|--------|------|
| `/api/status` | GET | Fleet status |
| `/api/health` | GET | System health check |
| `/api/trigger` | POST | Dispatch task via webhook |
| `/api/tasks/waiting-human` | GET | Agent HITL questions |
| `/api/tasks/<id>/respond` | POST | Answer agent question |
| `/api/tasks/queue` | GET | Pending task queue |
| `/api/agents/performance` | GET | Agent metrics |
| `/api/usage?period=day` | GET | Token spend |
| `/api/feedback` | POST | Approve/reject agent output |
